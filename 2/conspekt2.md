# Лекция: линейные коды, матрицы \(G\) и \(H\), коды Хэмминга и их дуалы

## 1. Организационные моменты

Рекомендация: составить график выступлений на месяц, чтобы все успели выступить хотя бы раз.

## 2. Понятие кода и линейные коды

Код — это **набор** кодовых слов фиксированной длины \(n\). Источник обычно считается равновероятным: все информационные слова длины \(k\) (в двоичном случае \(2^k\) комбинаций) считаются возможными.

Эффективный код должен иметь достаточно большую длину \(n\), чтобы обеспечить большое минимальное расстояние. Но рост \(n\) усложняет кодер и декодер, поэтому ищут компромисс.

Линейный \((n,k)\)-код — это линейное подпространство пространства всех векторов длины \(n\) над GF(2). Такой код можно описать порождающей матрицей \(G\); кодовые слова — все линейные комбинации строк \(G\).

---

## 3. Порождающая матрица \(G\)

Для \((n,k)\)-кода матрица \(G\) имеет размер \(k \times n\). Информационное слово \(m = (m_1,\dots,m_k)\) умножается на \(G\): \(c = mG\), и на выходе получаем кодовое слово \(c = (c_1,\dots,c_n)\).

Строки \(G\) линейно независимы. Ранг по строкам и по столбцам совпадает, значит существуют \(k\) линейно независимых столбцов.

---

## 4. Проверочная матрица \(H\), пространство проверок

Вектор \(h\) длины \(n\) называется проверкой (parity-check vector), если для любого кодового слова \(c\) выполняется \(c \cdot h = 0\) (скалярное произведение по GF(2)). Собирая проверки в строки, получаем матрицу \(H\) такую, что \(GH^{T} = 0\). У \(G\) размер \(k \times n\), у \(H\) — \((n-k) \times n\).

Ранг \(G\) равен \(k\), значит размерность ортогонального подпространства (пространства проверок) равна \(n - k\). Поэтому матрица \(H\) имеет \(n-k\) линейно независимых строк и размер \((n-k) \times n\).

Можно переставить столбцы \(G\) так, чтобы первые \(k\) линейно независимых составили информационную совокупность, а остальные — проверочную.

---

## 5. Систематический вид кода

\(G\) можно эквивалентными преобразованиями привести к виду \([I_k \; P]\), где \(I_k\) — единичная матрица, \(P\) — некоторая матрица. Код с такой \(G\) называется систематическим: первые \(k\) позиций кодового слова совпадают с информационным словом.

Для систематического \(G = [I_k\; P]\) проверочная матрица может быть взята в виде \(H = [P^{T}\; I_{n-k}]\).

Перестановки столбцов, сложение строк и т.п. дают эквивалентные коды — параметры \((n,k,d)\) при этом не меняются. 

---

## 6. Минимальное расстояние через \(G\) и через \(H\)

Через \(G\): минимальное расстояние \(d\) линейного кода равно минимальному весу ненулевого кодового слова: \(d = \min_{m \neq 0} \mathrm{wt}(mG)\). Чтобы вычислить его в лоб, нужно перебрать \(2^k - 1\) ненулевых информационных слов.

При скорости \(R = k/n > 1/2\) имеем \(k > n-k\), то есть \(H\) «короче» по числу строк, чем \(G\) — это даёт идею ускорения. Если \(cH^{T} = 0\) и \(\mathrm{wt}(c) = w\), то сумма \(w\) столбцов \(H\) равна нулю. Значит, минимальное расстояние — это минимальное число столбцов \(H\), линейно зависимых (сумма которых даёт 0), не считая тривиального случая.

**Теорема.** Минимальное расстояние линейного \((n,k)\)-кода равно \(d\) тогда и только тогда, когда любые \(d-1\) столбцов \(H\) линейно независимы, но существует набор из \(d\) линейно зависимых столбцов.

Ранг \(H\) равен \(n-k\), поэтому в \(H\) не может быть более \(n-k\) линейно независимых столбцов. Отсюда граница: \(d \leq (n-k) + 1\).

---

## 7. Дуальный код

Дуальный код — это код, у которого порождающая матрица совпадает с проверочной матрицей исходного кода. Если у исходного кода \(G_1, H_1\), то у дуального: \(G_2 = H_1\), \(H_2 = G_1\).

---

## 8. Пример: код с проверкой на чётность

Рассмотрим \((n, n-1)\)-код, у которого \(H\) — одна строка из всех единиц. Порождающую матрицу \(G\) можно записать как \([I_{n-1} \; \mathbf{1}]\), где последний столбец — все единицы.

Минимальное расстояние \(d = 2\): каждое ненулевое кодовое слово имеет чётный вес, минимальный — 2. Код не исправляет ни одной ошибки (для исправления одной нужен \(d \geq 3\)), но обнаруживает любые ошибки нечётного веса: любое кодовое слово имеет чётный вес, и добавление ошибки нечётного веса делает вес нечётным. Это код с проверкой на чётность (parity-check code). 

---

## 9. Коды Хэмминга: построение через \(H\)

Задача — построить код, исправляющий любые одиночные ошибки (вес ошибки \(=1\)).

Пусть \(e\) — вектор ошибки веса 1, а \(r = c + e\) — принятый вектор. Синдром: \(s = rH^{T} = cH^{T} + eH^{T} = 0 + eH^{T}\). При весе 1 вектора \(e\) выражение \(eH^{T}\) просто выбирает один столбец \(H\); синдром совпадает с одним из столбцов. Значит, если столбец \(H\) в позиции \(i\) равен синдрому, ошибка произошла в \(i\)-й позиции.

Идея: берём \(H\) размера \(r \times n\), записываем все ненулевые двоичные векторы длины \(r\) как столбцы \(H\), и считаем двоичный столбец номером позиции в двоичной записи. При \(r = 3\) столбцы: 001 (1), 010 (2), 011 (3), 100 (4), 101 (5), 110 (6), 111 (7). Тогда синдром напрямую кодирует номер позиции ошибки.

Число столбцов \(H\) (и длина кода) равно числу ненулевых векторов длины \(r\): \(n = 2^{r} - 1\). Размерность \(k\) получается из \(n-k = r\): \(k = n - r = 2^{r} - 1 - r\). Это семейство двоичных кодов Хэмминга \((2^{r}-1,\; 2^{r}-1-r)\).

Минимальное расстояние \(d = 3\), код исправляет одну ошибку. Двоичные коды Хэмминга оптимальны: при данных \(n\) и \(d=3\) не существует (даже нелинейных) кодов с большим числом кодовых слов. 

---

## 10. Симплексный код — дуальный коду Хэмминга

В проверочной матрице кода Хэмминга в каждой строке одинаковое число единиц. Для \(r\) строк это число равно \(2^{r-1}\) (каждый координатный бит «1» для половины всех ненулевых наборов).

Если взять \(G = H\) от кода Хэмминга, получаем симплексный код. Все кодовые слова дуального кода имеют один и тот же вес \(2^{r-1}\), а расстояние между любыми двумя кодовыми словами одинаково. Минимальное расстояние дуального кода: \(d = 2^{r-1}\). 

---

## 11. Расширенный код Хэмминга

Берём исходный код Хэмминга, например \((7,4)\), и его \(H\). Добавляем к \(H\): сначала нулевой столбец (все нули), затем строку из всех единиц. Получаем новую проверочную матрицу расширенного кода.

Для исходного кода Хэмминга: \(n = 2^{r}-1\), \(k = 2^{r}-1-r\), \(H\) — \(r \times n\). После расширения длина увеличивается на 1: \(n_{\text{new}} = 2^{r}\), размерность остаётся прежней: \(k_{\text{new}} = k\), то есть \((2^{r},\; 2^{r}-1-r)\).

Нулевой столбец просто удлиняет кодовые слова, не меняя расстояние. Добавление строки из единиц — это общая проверка на чётность; она увеличивает минимальное расстояние с 3 до 4. В расширенном коде Хэмминга \(d = 4\): можно исправить одну ошибку и дополнительно обнаружить некоторые ошибки веса 2. 

---

## 12. Дуальный к расширенному коду Хэмминга: код Рида–Маллера

К расширенному коду Хэмминга можно снова взять дуальный код: порождающая матрица нового кода — это проверочная матрица расширенного. Этот код называется кодом Рида–Маллера первого порядка.
