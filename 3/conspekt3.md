## 2. Напоминание: линейные коды, \(G\), \(H\), канал ДСК

Рассматриваем линейные \((n,k)\)-коды над полем GF(2) (определения переносятся и на GF(q)). Кодовые слова образуют линейное подпространство векторов длины \(n\); сумма двух кодовых слов снова кодовое слово. Скорость кода \(R = k/n\).

Порождающая матрица \(G\) имеет размер \(k \times n\), проверочная \(H\) = \((n-k) \times n\); выполняется \(GH^{T} = 0\) (нулевая матрица размера \(k \times (n-k)\)). При перестановке \(G\) и \(H\) местами получаем дуальный код.

Канал — двоичный симметричный (ДСК): каждый бит меняется с вероятностью \(p\) и остаётся без изменения с вероятностью \(1-p\). Задача декодера: по принятому слову \(y\) найти кодовое слово \(c\), отличающееся от \(y\) в наименьшем числе позиций (декодирование по максимуму правдоподобия в ДСК).

---

## 3. Вектор ошибки и количество комбинаций ошибок

Передано кодовое слово \(c\), в канале добавилась ошибка \(e\), принято \(y = c + e\) (сложение по модулю 2). Вектор \(e = (e_1,\dots,e_n)\) называется вектором ошибки. Всего векторов длины \(n\) — \(2^n\); это все возможные комбинации ошибок, включая \(e = 0\).

Если взять фиксированный \(e\) и прибавить его ко всем кодовым словам \(c\), получим множество \(\{c + e\}\) из \(2^k\) различных векторов (повторов внутри набора быть не может). Для разных ошибок эти множества устроены одинаково — это классы эквивалентности; таких классов ровно \(2^{n-k}\) (все \(2^n\) слов делятся на классы размера \(2^k\)). По сути это **косеты** (смещения) кодового подпространства, и каждый класс соответствует одному синдрому.

---

## 4. Синдромное декодирование: идея

Синдром \(s(y)\) для принятого слова \(y\) определяется как \(s = yH^{T}\). Для кодового слова \(c\) выполняется \(cH^{T} = 0\).

Связь с ошибкой: \(y = c + e \Rightarrow s(y) = yH^{T} = (c+e)H^{T} = cH^{T} + eH^{T} = eH^{T}\). Синдром зависит только от вектора ошибки, а не от конкретного кодового слова.

Для построения таблицы синдромов для всех ошибок \(e\) (на практике — ограниченного веса) вычисляют \(s = eH^{T}\). Ошибки с одинаковым синдромом образуют класс; из него выбирают представителя минимального веса — **наиболее вероятную** ошибку для этого синдрома.

**Алгоритм декодирования:** получили \(y\) → вычисляем синдром \(s = yH^{T}\) → по таблице находим вектор ошибки \(\hat e\) минимального веса → восстанавливаем кодовое слово: \(\hat c = y + \hat e\).

## 5. Небольшие примеры (устно в лекции)

### 5.1. Пример с кодом проверки на чётность (неудачный для исправления)

Порождающая \(G\) для \(n=3\): \([I_2 \mid \mathbf{1}]\) (код с проверкой на чётность), проверочная \(H = (1\;1\;1)\). Кодовые слова: 000, 101, 011, 110 (все чётного веса). Ошибки — все 8 векторов длины 3 от 000 до 111.

Минимальное расстояние \(d=2\), код ошибки не исправляет (например, 001 равноудалено от нескольких кодовых слов). Пример не подходит для однозначного исправления, но пригоден для иллюстрации классов эквивалентности: векторы группируются по чётности.

### 5.2. Пример с кодом, исправляющим одну ошибку

Берётся код с минимальным расстоянием 3 (например, простейший \((3,1)\)-код) и его \(H\). Строится таблица: для всех ошибок \(e\) от 000 до 111 считают \(s = eH^{T}\); для каждого синдрома выбирают ошибку веса 0 или 1 (код исправляет одну ошибку). При декодировании по пришедшему \(y\) по синдрому выбирается ошибка веса 1 и исправляется.

## 6. Радиус покрытия и сферы Хэмминга

Минимальное расстояние \(d_{\min}\) задаёт гарантированное число исправляемых ошибок: \(t = \left\lfloor\frac{d_{\min}-1}{2}\right\rfloor\). **Сфера Хэмминга** радиуса \(t\) с центром в кодовом слове \(c\) — множество всех векторов на расстоянии от \(c\) не более \(t\).

**Радиус покрытия** линейного \((n,k)\)-кода — наибольший вес вектора, который декодируется в нулевое кодовое слово (из-за линейности то же верно для любого центра). Геометрически это максимальный \(r\), при котором сферы радиуса \(r\) вокруг кодовых слов покрывают всё пространство (могут пересекаться).

Построение кода с заданным \(t\) можно трактовать как упаковку пространства длины \(n\) сферами радиуса \(t\) с центрами в кодовых словах. Идеальный случай — плотная упаковка: сферы покрывают пространство без «дыр» и без пересечений. Такие нетривиальные линейные коды (достигающие границы Хэмминга) называются **совершенными**. Примеры: двоичные коды Хэмминга и двоичный код Голэя.

## 7. Граница Хэмминга

Для \(q\)-ичного кода длины \(n\) с минимальным расстоянием \(d \ge 2t + 1\) число кодовых слов \(M\) удовлетворяет:
$$
M \cdot \sum_{i=0}^{t} \binom{n}{i}(q-1)^i \le q^n,
$$
или эквивалентно
$$
M \le \frac{q^n}{\sum_{i=0}^{t} \binom{n}{i}(q-1)^i},
$$
где \(t = \left\lfloor\frac{d-1}{2}\right\rfloor\).

Смысл: \(\sum_{i=0}^{t} \binom{n}{i}(q-1)^i\) — объём сферы Хэмминга радиуса \(t\) в \(q\)-ичном пространстве длины \(n\), \(q^n\) — общее число векторов. Сферы вокруг разных кодовых слов не пересекаются, поэтому суммарный «объём» не превосходит \(q^n\). Для двоичного кода (\(q=2\)) объём сферы: \(\sum_{i=0}^{t} \binom{n}{i}\).

Равенство в границе Хэмминга для нетривиальных линейных кодов достигается только кодами Хэмминга и кодами Голэя — они совершенные.

## 8. Граница Варшамова–Гильберта

Если для параметров \((n,k,d)\) выполняется
$$
q^{\,n-k} \ge \sum_{i=0}^{d-2} \binom{n-1}{i}(q-1)^i,
$$
то существует \(q\)-ичный линейный \((n,k,d)\)-код с таким минимальным расстоянием.

**Идея доказательства:** проверочную матрицу \(H\) строят по столбцам. Первый столбец — любой ненулевой; каждый следующий выбирают так, чтобы не возникало линейных комбинаций от 1 до \(d-2\) уже выбранных столбцов (т.е. любые \(d-1\) столбцов линейно независимы). При \((n-k)\) строках возможно \(q^{n-k}\) различных столбцов; «запрещённых» комбинаций не больше суммы в правой части неравенства. Если допустимых столбцов хватает, \(H\) нужной длины достраивается — код с заданным \(d\) существует.

Граница Хэмминга даёт верхнюю оценку на число кодовых слов, граница Варшамова–Гильберта — нижнюю, гарантируя существование кодов с заданным минимальным расстоянием.
