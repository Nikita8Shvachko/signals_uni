## 2. О чём курс

Курс называется «Обработка и интерпретация сигналов», но фактически акцент на **теории передачи данных через зашумлённые каналы** — канальном кодировании. Это не только телефон или радиоканал: те же идеи работают при чтении с флешки или жёсткого диска, при хранении данных в памяти.

Везде данные физически хранятся и передаются в аналоговом мире, а перед этим **кодируются** с добавлением избыточности — чтобы можно было обнаруживать и исправлять ошибки.

Работаем в основном с **дискретными (цифровыми) последовательностями**, в базовой модели — двоичная последовательность битов (0 и 1).

---

## 3. Упрощённая модель цифровой системы связи

1. **Источник данных** — генерирует информационные символы (в базовом случае биты).
2. **Кодер источника (канальный кодер)** — преобразует информационную последовательность в кодовую. Каждому информационному слову (или их группе) сопоставляет кодовое слово. В кодовом слове есть **избыточность**, позволяющая бороться с ошибками.
3. **Канал связи с шумом** — на вход подаются кодовые символы, внутри действует шум (ошибки), на выходе — искажённые кодовые символы (кодовое слово плюс ошибки).
4. **Декодер источника (канальный декодер)** — по принятому зашумлённому слову пытается восстановить исходное кодовое слово, а затем и информационные символы.
5. **Получатель** — получает оценку исходной последовательности (надеемся, совпадает с тем, что было на выходе источника).

---

## 4. Математическая модель канала: двоичный симметричный канал (ДСК)

Работаем с **двоичным симметричным каналом**. На вход по одному подаются биты (0 или 1). На выходе каждый бит передаётся **правильно** с вероятностью 1 - p и **меняется** на противоположный с вероятностью p. Величина p — **переходная вероятность**, она же «вероятность ошибки одного символа».

Словами: вход 0 остаётся 0 с вероятностью 1-p и превращается в 1 с вероятностью p; вход 1 — наоборот.

Пример: p = 10^{-3}. При передаче порядка 1000 бит ожидается около одной ошибки.

---

## 5. Идея улучшения надёжности: дублирование

Если передавать биты **без кодирования**, вероятность ошибки на выходе канала равна p. Чтобы её уменьшить, можно **дублировать данные**.

### Простой пример: повторный код (3,1)

Информационный бит кодируется так: 0 \mapsto 000, 1 \mapsto 111. Кодовые слова длины 3: K = 1 информационный символ, N = 3, скорость R = K/N = 1/3.

Декодирование **по большинству**: 000, 001, 010, 100 → принимаем 0; 111, 110, 101, 011 → принимаем 1. Код исправляет **одну ошибку** в тройке бит.

Вероятность ошибки декодирования (для одного информационного бита) — только когда в тройке возникло **минимум две ошибки**: 2 ошибки с вероятностью 3 p^{2}(1-p), 3 ошибки с вероятностью p^{3}. Итого:
$$
P_{\text{err, код}} = 3p^{2}(1-p) + p^{3}.
$$
Для p = 10^{-3} это уже существенно меньше, чем p. Цена: скорость передачи уменьшилась в 3 раза.

---

## 6. Более «экономичный» код: пример (5,2)

Вместо кодирования по одному биту кодируем пары. Информационные слова (2 бита): 00, 01, 10, 11. Им сопоставляются кодовые слова длиной 5 (пример с лекции): 00 → 00000, 01 → 10110, 10 → 01011, 11 → 11101.

Код тоже исправляет **одну ошибку** (по правилу наименьшего расстояния). Параметры: K = 2, N = 5, скорость R = 2/5 = 0.4.

Сравнение: повторный код (3,1) даёт R = 1/3 \approx 0.333, код (5,2) — R = 0.4 > 1/3. Оба исправляют 1 ошибку, но второй даёт **более высокую скорость** ⇒ он «лучше» в смысле эффективности использования канала.

---

## 7. Блоковые коды и скорость

Рассматриваются **блоковые коды**: информационная последовательность режется на блоки длины K (информационные слова), каждый блок кодируется в кодовое слово длины N.

Обозначения: K — число информационных символов в блоке, N — длина кодового слова. **Скорость кода** R = K/N — сколько информационных битов приходится на один передаваемый бит.

Примеры: без кодирования R = 1, повторный код R = 1/3, код (5,2) — R = 2/5.

## 8. Теорема Шеннона и пропускная способность канала

Клод Шеннон в 1948 г. ввёл ключевые понятия.

### Энтропия двоичного ансамбля

Для p \in [0,1]:
$$
h(p) = -p \log_2 p - (1-p)\log_2 (1-p).
$$

### Пропускная способность двоичного симметричного канала

$$
C = 1 - h(p).
$$

Здесь p — вероятность ошибки одного символа в ДСК, C — **максимальная достижимая скорость передачи информации** (в битах на 1 переданный бит), при которой с помощью подходящих кодов можно достигать сколь угодно малой вероятности ошибки.

**Результат Шеннона.** Если R < C, существует (теоретически) код длины N при достаточно большом N, для которого вероятность ошибки декодирования сколь угодно мала. Если R > C, **надёжная передача невозможна**: вероятность ошибки ограничена снизу, как бы мы ни усложняли код.

Пример для p = 10^{-3}: по вычислению на лекции C \approx 0.9888252. То есть максимальная скорость всё ещё почти 1 — достаточно добавить около **2% избыточности**, чтобы (теоретически) сделать вероятность ошибки как угодно маленькой.

Проблема: доказательство Шеннона **неконструктивно** — он доказывает существование таких кодов, но не показывает, как их построить. Реальные коды (LDPC, turbo и др.) приближаются к пределу, но полностью его не достигают.

## 9. Вес Хэмминга и расстояние Хэмминга

Работаем в двоичном случае (биты 0 и 1).

### Вес Хэмминга

Для кодового слова x (двоичный вектор) вес Хэмминга \omega(x) — это число ненулевых элементов, в двоичном случае просто **число единиц**. Пример: x = 01011 → \omega(x) = 3.

### Расстояние Хэмминга

Расстояние Хэмминга d(x,y) между двумя словами — число позиций, в которых они отличаются. Пример: x = 01011, y = 11001 отличаются в 3 позициях → d(x,y)=3.

В двоичном случае (сложение по модулю 2):
$$
d(x,y) = \omega(x + y),
$$
где x + y — поразрядное сложение. Также d(x,0) = \omega(x).

### Минимальное расстояние кода

Пусть C — множество всех кодовых слов. Минимальное расстояние:
$$
d_{\min} = \min_{x \neq y, x,y \in C} d(x,y).
$$
Это «слабое место» кода — самые близкие друг к другу слова.

Для **линейного двоичного кода** есть упрощение: сумма любых двух кодовых слов тоже кодовое слово, поэтому
$$
d_{\min} = \min_{c \in C, c \neq 0} \omega(c),
$$
то есть минимальное расстояние равно **минимальному ненулевому весу** кодового слова.

### Связь d_{\min} и числа исправляемых ошибок

Если код исправляет все ошибки кратности t (любые t ошибок в блоке длины N), то
$$
t \le \left\lfloor \frac{d_{\min} - 1}{2} \right\rfloor.
$$
Интуиция: «окрестности» радиуса t вокруг кодовых слов не должны пересекаться; минимальное расстояние между центрами этих «шаров» — d_{\min}.

## 10. Линейные коды

### Определение линейного кода (двоичный случай)

**Линейный код** — код, у которого множество кодовых слов C образует **линейное подпространство** над GF(2): если x, y \in C, то и x + y \in C (сложение по модулю 2), есть нулевое слово, всякие линейные комбинации кодовых слов тоже лежат в C.

В двоичном случае работаем в пространстве GF(2)^N. Параметры линейного [N,K]-кода: N — длина кодового слова, K — размерность подпространства (число «независимых информационных координат»), скорость R = K/N.

### Линейный q-ичный код

Обобщение: над полем GF(q) линейным [N,K]-кодом называют **K-мерное линейное подпространство** пространства GF(q)^N. Число кодовых слов — q^K.

---

## 11. Порождающая матрица

Множество кодовых слов линейного кода — подпространство, у него есть **базис**. Пусть C — линейный [N,K]-код над GF(q). Возьмём базис в виде K векторов длины N — это будут строки матрицы G.

**Порождающая матрица** G: размер K \times N, строки — базисные векторы кода. Любое кодовое слово записывается как c = m G, где m = (m_1, \dots, m_K) — **информационное слово**, умножение и сложение — по правилам поля GF(q).

Для двоичного кода m — двоичный вектор длины K, G — матрица над GF(2); при переборе всех 2^K возможных m получаем все кодовые слова.

---

## 12. Проверочная матрица

Идея: построить набор векторов, **ортогональных** всем кодовым словам. Вектор H длины N такой, что для любого кодового слова c выполняется \langle c, H \rangle = 0 (скалярное произведение в GF(2): поразрядное произведение и сумма).

Собрав несколько таких независимых векторов, получаем **проверочную матрицу** H размером (N-K) \times N; её строки — проверки, ортогональные подпространству кода.

Свойства: G H^{T} = 0; для любого кодового слова c выполняется c H^{T} = 0. Для произвольного вектора r величина s = r H^{T} называется **синдромом** — по нему можно пытаться определить, какие ошибки произошли.
